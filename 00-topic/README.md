# Topic Selection

## Reason for Choosing This Topic

This topic was selected because a mini mart is a familiar business model with clear daily operations. A small retail store needs to manage products, customers, employees, invoices, payments, and stock quantity. These operations are suitable for demonstrating a relational database design.

The topic is also practical for a DBI202 assignment because it includes:

- Many related entities.
- One-to-many relationships.
- Transaction records through invoices and invoice details.
- Inventory updates after sales.
- Business reports using SQL queries.
- Opportunities to use stored procedures, functions, and triggers.

## Problem Statement

In many small stores, sales and inventory are still recorded manually. This can lead to incorrect stock counts, missing sales information, difficulty checking revenue, and slow report generation.

The Mini Mart Sales Management System helps solve these problems by storing sales and inventory data in a structured database.

## Project Goal

The goal of this assignment is to design and demonstrate a database system that supports basic mini mart operations, including product management, sales invoice management, payment tracking, inventory control, and business reporting.
